JaxAHT organizes its algorithms into four directories — `marl/`, `ego_agent_training/`, `teammate_generation/`, and `open_ended_training/` — each with a `run.py` entry point. All algorithms use [Hydra](https://hydra.cc/) for configuration, so you can swap tasks, algorithms, and hyperparameters entirely from the command line without editing any files. This guide walks you through the three most common workflows to get you oriented quickly.

<Note>
  Complete installation before proceeding. If you have not yet installed JaxAHT and downloaded the evaluation data, start with the [installation guide](/installation).
</Note>

## Run IPPO on Level-Based Foraging

IPPO (Independent PPO with parameter sharing) is the MARL algorithm in JaxAHT and also the backbone for teammate generation methods like FCP. Running it on LBF is the fastest way to confirm your setup is working end to end.

<Steps>
  <Step title="Launch the training run">
    From the repository root, run:

    ```bash
    python marl/run.py task=lbf/lbf_7x7_nolevels algorithm=ippo/lbf/lbf_7x7_nolevels
    ```

    Hydra merges the task config (`configs/task/lbf/lbf_7x7_nolevels.yaml`) and the algorithm config (`configs/algorithm/ippo/lbf/lbf_7x7_nolevels.yaml`) into the master config at `marl/configs/base_config_marl.yaml`, then prints the resolved config before training starts.
  </Step>

  <Step title="Monitor training metrics">
    Results are written to two places by default:

    - **Local filesystem** — `results/lbf/lbf_7x7_nolevels/ippo/` (as configured in `configs/hydra/hydra_simple.yaml`)
    - **Weights & Biases** — the project specified in the master config

    <Tip>
      The maximum achievable return on the `lbf_7x7_nolevels` task is **0.5**. Use this as a reference to judge training progress. The `returned_episode_returns` metric tracked by the `LogWrapper` reflects the sum of shaped rewards per episode, not a normalized score.
    </Tip>

    To run with W&B logging disabled (useful for quick local tests), pass `logger.mode=offline`:

    ```bash
    python marl/run.py task=lbf/lbf_7x7_nolevels algorithm=ippo/lbf/lbf_7x7_nolevels logger.mode=offline
    ```
  </Step>

  <Step title="Locate the saved checkpoint">
    After training completes, find the checkpoint directory for use in downstream workflows:

    ```python
    import glob, os

    checkpoint_base_dir = "results/lbf/lbf_7x7_nolevels/ippo/default_label/"
    checkpoint_dir = sorted(glob.glob(os.path.join(checkpoint_base_dir, "*")))[-1]
    checkpoint_dir = os.path.join(checkpoint_dir, "saved_train_run")
    print(checkpoint_dir)
    ```

    You will use this path in the next section when training an ego agent against these teammates.
  </Step>
</Steps>

## Train an ego agent

Ego agent training algorithms in `ego_agent_training/` train an AHT agent (the "ego") against a fixed population of partner agents. The ego agent training entry point requires you to specify a partner config that points to the teammates the ego will train against.

<Steps>
  <Step title="Choose an algorithm and task">
    JaxAHT provides three ego agent training algorithms. Use the `algorithm=` argument to select one:

    <CodeGroup>

    ```bash PPO ego (LBF)
    python ego_agent_training/run.py \
      task=lbf/lbf_7x7_nolevels \
      algorithm=ppo_ego/lbf/lbf_7x7_nolevels
    ```

    ```bash LIAM ego (LBF)
    python ego_agent_training/run.py \
      task=lbf/lbf_7x7_nolevels \
      algorithm=liam_ego/lbf/lbf_7x7_nolevels
    ```

    ```bash MeLIBA ego (LBF)
    python ego_agent_training/run.py \
      task=lbf/lbf_7x7_nolevels \
      algorithm=meliba_ego/lbf/lbf_7x7_nolevels
    ```

    </CodeGroup>
  </Step>

  <Step title="Specify the partner checkpoint">
    The partner config tells the ego agent training algorithm where to find the teammates. Pass the checkpoint path from the IPPO run above directly on the command line using Hydra's dot-notation override syntax:

    ```bash
    python ego_agent_training/run.py \
      task=lbf/lbf_7x7_nolevels \
      algorithm=ppo_ego/lbf/lbf_7x7_nolevels \
      algorithm.partner_agent.ippo.path=<checkpoint_dir>
    ```

    Replace `<checkpoint_dir>` with the full path printed in the previous section (e.g., `results/lbf/lbf_7x7_nolevels/ippo/default_label/<run_id>/saved_train_run`).

    <Note>
      The ego agent training entry point runs a heldout evaluation against the downloaded evaluation teammates at the end of training by default. You can disable this with `run_heldout_eval=false` to speed up iteration during development.
    </Note>
  </Step>

  <Step title="Reduce training steps for a quick test">
    To verify the pipeline without waiting for a full training run, override `TOTAL_TIMESTEPS`:

    ```bash
    python ego_agent_training/run.py \
      task=lbf/lbf_7x7_nolevels \
      algorithm=ppo_ego/lbf/lbf_7x7_nolevels \
      algorithm.partner_agent.ippo.path=<checkpoint_dir> \
      algorithm.TOTAL_TIMESTEPS=1e6 \
      run_heldout_eval=false \
      logger.mode=offline
    ```
  </Step>
</Steps>

## Generate teammates with FCP

Teammate generation algorithms in `teammate_generation/` produce diverse agent populations for use as training partners. FCP (Fictitious Co-Play) generates this diversity by training IPPO with varying random seeds and saving intermediate checkpoints.

<Steps>
  <Step title="Run FCP on LBF">
    ```bash
    python teammate_generation/run.py \
      task=lbf/lbf_7x7_nolevels \
      algorithm=fcp/lbf/lbf_7x7_nolevels
    ```

    Hydra loads `configs/base_config_teammate.yaml` as the master config and merges in the FCP algorithm config and LBF task config.
  </Step>

  <Step title="Choose a different teammate generation algorithm">
    Swap `fcp` for any other algorithm by changing the `algorithm=` key:

    <CodeGroup>

    ```bash FCP
    python teammate_generation/run.py \
      task=lbf/lbf_7x7_nolevels \
      algorithm=fcp/lbf/lbf_7x7_nolevels
    ```

    ```bash BRDiv
    python teammate_generation/run.py \
      task=lbf/lbf_7x7_nolevels \
      algorithm=brdiv/lbf/lbf_7x7_nolevels
    ```

    ```bash LBRDiv
    python teammate_generation/run.py \
      task=lbf/lbf_7x7_nolevels \
      algorithm=lbrdiv/lbf/lbf_7x7_nolevels
    ```

    ```bash CoMeDi
    python teammate_generation/run.py \
      task=lbf/lbf_7x7_nolevels \
      algorithm=comedi/lbf/lbf_7x7_nolevels
    ```

    </CodeGroup>
  </Step>

  <Step title="Run with an ego agent and heldout evaluation">
    Teammate generation methods can optionally train an ego agent against the generated population and then evaluate it on the heldout set, all in one command. Both flags are set in the master config and can be overridden from the command line:

    ```bash
    python teammate_generation/run.py \
      task=lbf/lbf_7x7_nolevels \
      algorithm=fcp/lbf/lbf_7x7_nolevels \
      train_ego=true \
      run_heldout_eval=true
    ```
  </Step>
</Steps>

## Override config values from the command line

All JaxAHT algorithms use Hydra, so any value in the resolved config tree can be changed from the command line without editing a YAML file. This makes it easy to sweep hyperparameters or run quick experiments.

```bash
# Set total training timesteps for FCP
python teammate_generation/run.py \
  task=lbf/lbf_7x7_nolevels \
  algorithm=fcp/lbf/lbf_7x7_nolevels \
  algorithm.TOTAL_TIMESTEPS=1e5

# Run on a different task (Overcooked cramped_room)
python marl/run.py \
  task=overcooked-v1/cramped_room \
  algorithm=ippo/overcooked-v1/cramped_room

# Disable W&B and reduce timesteps for a smoke test
python marl/run.py \
  task=lbf/lbf_7x7_nolevels \
  algorithm=ippo/lbf/lbf_7x7_nolevels \
  algorithm.TOTAL_TIMESTEPS=5e4 \
  logger.mode=offline
```

<Tip>
  You can inspect the full resolved config before training begins — Hydra prints it to the console at startup. Use this to confirm your overrides took effect before committing to a long run.
</Tip>
